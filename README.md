# Ads Metrics Calculator

**Ads Metrics Calculator** — это быстрый и воспроизводимый CLI-инструмент (интерфейс командной строки), разработанный для медиабайеров, арбитражников и дата-аналитиков. Он обрабатывает сырые данные о трафике из различных партнерских сетей и мгновенно рассчитывает ключевые показатели эффективности (KPI), такие как ROI, Конверсия (CR) и Чистая прибыль (Net Profit).

### 🐳 Быстрый запуск через Docker
Приложение полностью контейнеризировано! Вы можете запустить его локально без установки Python или настройки окружения:
```bash
make compose-up
```
*(Никакой возни с базами данных или конфликтами зависимостей)*

## Какую проблему мы решаем
Работа с данными о трафике из множества партнерских сетей (например, Leadgid, Mitgo, Shakes.pro) часто сводится к работе с сырыми JSON-выгрузками. Расчет метрик вручную в Excel отнимает много времени и чреват ошибками. **Ads Metrics Calculator** решает эту проблему путем жесткого разделения логики парсинга сырых данных и чистой математической бизнес-логики. Это гарантирует безопасность расчетов и корректную обработку граничных случаев, таких как нулевые клики или аномалии нулевой стоимости.

## Ключевые особенности
* **Строгие архитектурные границы:** Бизнес-логика полностью изолирована от CLI-интерфейса, что соответствует лучшим практикам программной инженерии.
* **Встроенный AI-аудит (Grill-Me):** Интегрированный профиль ИИ-агента для жесткого стресс-тестирования архитектуры и инфраструктуры (CI/CD, Docker) по стандартам Clean Architecture.
* **Защита от граничных случаев:** Встроенные предохранители против деления на ноль, отрицательных финансовых значений и аномалий отслеживания.
* **Автоматизированное тестирование:** Исчерпывающее покрытие тестами, включая `smoke` и `unit` тесты (с позитивными и негативными сценариями, а также проверками плавающей запятой) с использованием `unittest`.
* **Воспроизводимое окружение:** Полностью докеризированная среда выполнения с оптимизированным контекстом сборки.
* **Автоматизация рабочих процессов:** Единый интерфейс `Makefile` для сборки, тестирования и симуляции CI пайплайнов.

## Стек технологий
* **Бэкенд и логика:** Python 3.12 (Только стандартная библиотека, без сторонних зависимостей)
* **Тестирование:** `unittest`, `subprocess`, `coverage`
* **Инфраструктура:** Docker, Docker Compose
* **Автоматизация и Документация:** GNU Make, Sphinx, build, twine

## Как работают метрики
Мы не просто вслепую делим числа. Наш основной модуль (`packages/core/metrics.py`) применяет строгие математические правила:
* **ROI (Окупаемость инвестиций):** Расчет рентабельности. *Статус: Выбрасывает `ValueError` при отрицательных затратах или аномалиях нулевой стоимости, предотвращая ложные отчеты.*
* **CR (Коэффициент конверсии):** Оценка эффективности генерации лидов. *Статус: Безопасно возвращает `0.0`, если клики равны 0.*
* **Net Profit (Чистая прибыль):** Классический расчет `доход - расход` для мгновенной оценки финансового результата.

## Локальная установка и запуск

Чтобы запустить Ads Metrics Calculator на вашем компьютере, выполните следующие шаги:

**1. Клонируйте репозиторий**
```bash
git clone [https://github.com/Vadim077/project_ROI_metrics.git](https://github.com/Vadim077/project_ROI_metrics.git)
cd project_ROI_metrics
```

**2. Глубокая проверка инфраструктуры (CI Simulation)**
Убедитесь, что код, тесты, документация и сборка работают корректно, запустив симуляцию CI:
```bash
make check
```

**3. Запустите приложение локально (Нативно)**
Запустите скрипт, используя предоставленный тестовый файл с данными о трафике:
```bash
make run
```

**4. Запустите через Docker Compose (Рекомендуется)**
Выполните программу в полностью изолированном контейнере:
```bash
make compose-up
```

**5. Сборка и установка переиспользуемого ядра**
Соберите чистую бизнес-логику как независимый пакет с глобальной точкой входа:
```bash
make build-lib
make install-lib-local
```
*(После установки вы сможете использовать команду `ads-calc` из любой директории вашей системы).*

**6. Дополнительные команды разработки**
- Проверка покрытия тестами: `make coverage`
- Генерация HTML-документации: `make docs-sphinx`
- Публикация пакета (TestPyPI): `make publish-lib`
